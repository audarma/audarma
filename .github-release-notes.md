# Audarma v0.1.0-alpha.0 - Initial Alpha Release

This is the initial alpha release of **Audarma**, an LLM-powered translation system for React/Next.js applications.

## What is Audarma?

Audarma translates dynamic content (products, messages, user-generated content) using Large Language Models, with intelligent caching to minimize API costs. Unlike traditional i18n libraries for static UI labels, Audarma focuses on content that changes frequently and can't be pre-translated.

## Features

- **LLM-Powered Translation** - Use OpenAI, Anthropic Claude, Cerebras, or any LLM provider
- **View-Level Translation Tracking** - Translate entire views at once, not individual strings
- **Smart Caching** - Content hash tracking prevents unnecessary re-translations
- **Progressive Loading** - Shows original text immediately, translates in background
- **Adapter Pattern** - Bring your own database, LLM provider, and i18n library
- **Batch Translation** - Groups multiple items into single LLM calls
- **React Hooks** - Simple, composable API with loading states
- **Dual-Mode Operation** - Lazy (on-demand) + CLI (batch pre-translation)

## Included Adapters

### LLM Providers
- **OpenAI** - GPT-5, GPT-4.1, o4-mini
- **Anthropic** - Claude Sonnet 4.5, Haiku 4.5, Opus 4.1
- **Cerebras** - Qwen3-235B, DeepSeek R1 (free daily credits - ideal for demos!)
- **Nebius** - OpenAI-compatible API

### Database Adapters
- **Supabase** - PostgreSQL adapter

### I18n Adapters
- **next-intl** - Next.js i18n integration

## Installation

```bash
npm install audarma
```

## Quick Start

```typescript
import { AudarProvider, ViewTranslationProvider, useViewTranslation } from 'audarma';

// 1. Configure adapters
const config = {
  database: createSupabaseAdapter(supabase),
  llm: createOpenAIProvider({ apiKey: process.env.OPENAI_API_KEY }),
  i18n: { getCurrentLocale: () => locale, ... }
};

// 2. Wrap your app
<AudarProvider config={config}>
  {children}
</AudarProvider>

// 3. Use in components
const { text, isTranslating } = useViewTranslation('product_title', id, title);
```

## Known Limitations (Alpha Release)

This is an early extraction from a production app. Known limitations:

1. Hard-coded English as source language
2. No cache invalidation API
3. No error boundaries
4. No retry logic
5. No cost tracking
6. Client-side only (no server component support)
7. No streaming translations
8. No partial cache updates

See [full list of limitations](https://github.com/audarma/audarma#limitations--known-issues) and [roadmap](https://github.com/audarma/audarma#roadmap).
